# Roadmap: мобильный сайт с видео-контентом

Пошаговый план от нуля до полностью работающего продукта (без этапов тестирования и деплоя).

---

## 1. Подготовка проекта и базовая вёрстка

- [ ] Инициализировать проект (Vite + HTML/CSS/JS или выбранный стек)
- [ ] Настроить mobile-first viewport и базовые стили (тёмная тема по скринам)
- [ ] Подключить шрифты и CSS-переменные для цветов (фон, акцент — розовый/красный)
- [ ] Создать базовую HTML-структуру одной страницы с семантическими секциями

---

## 2. Первый экран: подтверждение 18+ и выбор типа контента

- [x] Реализовать полноэкранный первый экран (оверлей/отдельный экран до основного контента)
- [x] Блок подтверждения возраста: текст про 18+, кнопка «Да, мне есть 18» / «Войти» (подтверждение)
- [x] Без подтверждения 18+ основной контент не показывать (состояние/роут или скрытие секций)
- [x] Блок выбора типа контента: три варианта — тип 1, тип 2, тип 3 (кнопки или карточки)
- [x] Только один тип контента можно выбрать; визуально выделить выбранный (активное состояние)
- [x] После выбора типа контента — переход или показ основной страницы (шапка, видео, тарифы и т.д.)
- [x] Сохранить выбранный тип контента в состоянии (sessionStorage/localStorage или state), использовать для фильтрации или отображения контента
- [x] Доступность: понятные `aria-label`, фокус на кнопках подтверждения и выбора типа, возможность управления с клавиатуры

---

## 3. Шапка (Header)

- [x] Разметить шапку: логотип по центру (JAV + HD в рамке), тёмный фон
- [x] Добавить блок выбора языка/региона слева (иконка флага + стрелка)
- [x] Добавить иконку бургер-меню справа
- [x] Стилизовать шапку: фиксированная/липкая при скролле, высота, отступы
- [x] Реализовать доступность: `aria-label`, `tabindex` для интерактивных элементов

---

## 4. Блок «Выбранное видео»

- [x] Верстать блок с превью видео (изображение/плейсхолдер 16:9 или по дизайну)
- [x] Добавить иконки: выключенный звук и закрытие (крестик) в углу превью
- [x] Под превью: карточка с аватаркой канала, заголовком видео, тегами/названием канала
- [x] Кнопка «WATCH FULL VIDEO» (ярко-розовая), стили и состояние наведения
- [x] Подключить данные выбранного видео (заголовок, теги, канал) из разметки или JS

---

## 5. Страница с тарифами

- [x] Секция с заголовком типа «Join to get access to 1M+ videos»
- [x] Список тарифов: Trial, 1 month, 3 months, 12 months — радиокнопки или кастомные переключатели
- [x] Для каждого тарифа: цена в день, итоговая сумма и период (например 7 days for $4.83)
- [ ] Бейджи: «Limited offer», «-50% off», «+3 free months», «-67% off»
- [x] Визуально выделить выбранный тариф (активная радиокнопка, контраст)
- [ ] Горизонтальный скролл при необходимости (стрелки по краям по скрину)
- [x] Адаптивная сетка/список под маленькие экраны

---

## 6. Методы оплаты

- [ ] Блок под тарифами: CARD (выбрано), CRYPTOCOINS, Alipay, pay, OXXO, pix, UPI
- [ ] Реализовать выбор способа оплаты (кнопки/табы), активное состояние для CARD
- [ ] Стили: акцент для выбранного метода (розовый по скрину)

---

## 7. Блок «Другие видео для перехода»

- [x] Секция со списком/сеткой превью других видео
- [x] Горизонтальный скролл или сетка 2 колонки (mobile-first)
- [x] Для каждого видео: превью, название, теги, название канала
- [x] Клик по карточке — смена «выбранного видео» в блоке из п.4 (или переход на страницу видео)
- [x] Lazy loading для изображений превью (WebP, размеры, `loading="lazy"`)

---

## 8. Данные для каждого видео

- [ ] Определить структуру данных: id, название, теги[], канал, превью URL, ссылка на видео
- [ ] Заполнить или подключить источник данных (статичный JSON / API)
- [ ] Рендерить список «других видео» (п.7) и блок «выбранное видео» (п.4) из одних и тех же данных
- [ ] При выборе видео из списка — обновлять блок выбранного (название, теги, канал, превью)

---

## 9. Форма «Создать аккаунт / Войти»

- [x] Блок «Create new account or Sign In»
- [x] Поле ввода Email с плейсхолдером «Email address»
- [x] Валидация: показ «Email is required» при пустом отправлении (красный текст)
- [x] Кнопка «CREATE NEW ACCOUNT» (стиль как у CTA на скрине)
- [ ] Текст внизу с ссылками «Terms of use» и «Privacy policy»
- [x] Обработчики: `handleSubmit`, `handleEmailChange`; блокировка отправки при невалидном email

## 10. Бургер-меню

- [x] По клику на иконку бургера — открытие панели/оверлея меню
- [x] Закрытие по повторному клику, по клику вне панели или по клавише Escape
- [x] Разметка пунктов меню: Content removal, DMCA, For creators, Terms of use, Privacy policy, Refund policy, Affiliate program, Parental control, Contact us, DVD, Categories & tags, Create new account/Sign in/Become a member
- [x] Инфо-страницы в новой вкладке с кнопкой «Назад»; DVD и Categories — ссылки; Sign in/Create account — переход на страницу регистрации; Become a member при зарегистрированном пользователе
- [x] Анимация открытия/закрытия, доступность (focus trap, `aria-expanded`, `aria-label`)

## 11. Сборка страницы и интерактивность

- [x] Собрать все секции: Первый экран (18+ и тип контента) → после подтверждения: Header → Выбранное видео → Тарифы → Оплата → Другие видео → Форма
- [ ] Проверить порядок и отступы между блоками по скринам
- [x] Реализовать смену выбранного видео при клике по карточке из «других видео» (блок из п.4)
- [ ] Реализовать переключение тарифов и способов оплаты без перезагрузки
- [x] Убедиться, что форма не отправляет данные на сервер без бэкенда (предусмотреть заглушку или `preventDefault` с сообщением)
